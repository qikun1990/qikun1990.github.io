    Third thing was that I need something to promote.

    That already generated some money.

    Some of my torrents had up to 15,000 downloads or even more so there definitely was enough traffic to work with.
    But as you can imagine not all users who downloaded the torrents also visited my website.

    Then I thought I need to force them to come to my website so I started to upload passworded archives.
    In the folder with the archive was a link to my website and on the website the users then found the passwords for the downloaded torrents.

    That helped alot to drive more traffic to my websites.
    Additionally to the adult banners and Layer Ads I also added banners for a Usenet program to my website.

    I was running that setup for about 3 years or so.

    Up to this point I uploaded all torrents from my home PC and just used a VPN to encrypt my connection.

    When I started uploading the fake movies I got me a Windows VPS for seeding so I basically could run all programs there that I had on my home PC as well.

    I installed Winautomation on the VPS and started working on the automation part.

    For the movies I used a file pumper.
    This is a small software where you can create fake files with a size that you set.

    For the movie names I had one file with names of movie release crews and one file with names of the movies I wanted to use.

    To create the torrents I found a Pythin script so I also installed Python on the VPS.

    Then I created a ‚ÄúBot‚Äù in Winautomation that generated a random number between 700 and 770 or so.

    It also grabbed a random line from the release group file and renamed the fake file to MOVIENAME-RELEASEGROUP.avi

    Now the Winautomation bot generated a random and uncrackable few hundred character long password.
    To create the passworded archive it used the commandline version of WinRAR.

    When the archive was created the bot put a link to my website with name ‚ÄúGet the password‚Äù into the folder and created another archive from the folder that was not passworded.

    This archive then was used to run a commandline for the Python script to create a torrent from it.

    Additionally it created a new text file with the exact name of the ‚Äúrelease‚Äù that I can copy/paste it later to upload the torrents on the websites.

    I set the torrent client that way to automatically check my seed folder for new torrents and add them automatically to the client.

    These steps where looped for all lines in the moviename file.

    After it was done all I had to do was to upload the torrents to the torrent websites so that seeding started.

    To upload the torrents I only used very few websites to keep the workload there also as low as possible.
    From these few websites the torrents were grabbed from other websites anyway and got spread even more.
